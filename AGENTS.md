**Core Directive:** Maintain, update, and audit Victor's Second Brain Obsidian Vault to ensure high knowledge density, prevent repetition, and automate bookkeeping.

## 1. Core Architecture & Folder Conventions

The vault is structured according to the PARA method, evolved lightly with a hybrid approach to support raw sources:

- **[00-Inbox](file:///home/redmane/Documents/SecondBrain/00-Inbox)**: The entry point for temporary notes, quick thoughts, and unprocessed items.
- **[01-Projects](file:///home/redmane/Documents/SecondBrain/01-Projects)**: Active projects with specific deadlines/goals (e.g., `01-Projects/Momentum/`). Each project should use the **9-Doc Standard** when fully defined.
- **[02-Areas](file:///home/redmane/Documents/SecondBrain/02-Areas)**: Long-term responsibilities, health, finances, personal growth.
- **[03-Resources](file:///home/redmane/Documents/SecondBrain/03-Resources)**: Interests, topics, references, and external clips that don't belong to a specific area or project.
- **[04-Archive](file:///home/redmane/Documents/SecondBrain/04-Archive)**: Completed or inactive items.
- **[05-Daily](file:///home/redmane/Documents/SecondBrain/05-Daily)**: Periodic journals and logs.
- **[Clippings](file:///home/redmane/Documents/SecondBrain/Clippings)**: Folder for raw, immutable browser clips generated by `Obsidian-Web-Clipper`.
- **`raw/`** (Optional): A folder for immutable original source documents (PDFs, papers, logs) when strict immutability is desired.
- **`scripts/`**: Directory for local automation and utility scripts (e.g. `vault-librarian.js`).
- **`06-Agent-Sessions/`**: Repository for completed agent session logs and Q&A history logs.

### Rules for Editing
- **Never modify raw files**: Any file in [Clippings](file:///home/redmane/Documents/SecondBrain/Clippings) or `raw/` is immutable.
- **Keep index and changelog current**: Update [index.md](file:///home/redmane/Documents/SecondBrain/index.md) and [CHANGELOG.md](file:///home/redmane/Documents/SecondBrain/CHANGELOG.md) upon any structural modification.
- **Surgical edits**: Minimize changes to existing human-authored text; append or update specific sections, keeping one-line summaries at the top.
- **No Agent-Specific Hardcoding**: Do not write instructions or rules that assume only one specific AI client is running.
- **Empty Directory Placeholders**: Git does not track empty directories. Whenever creating a new empty folder that needs to be preserved in the repository, create a `.gitkeep` file inside it to ensure it is committed.

## 2. Operations & Workflows

### A. Ingest Workflow
*Triggered when the user requests an ingest or when new items appear in [00-Inbox](file:///home/redmane/Documents/SecondBrain/00-Inbox) or [Clippings](file:///home/redmane/Documents/SecondBrain/Clippings).*

1. **Monitor Clippings**: Proactively check the [Clippings](file:///home/redmane/Documents/SecondBrain/Clippings) folder for new items from `Obsidian-Web-Clipper`. When new clips appear, ingest them into PARA using `defuddle` if applicable.
2. **Read & Summarize**: Read the raw clipper note/source. Extract key takeaways.
3. **Check for Redundancies**: Search the vault (via index/grep) to see if this topic/problem is already addressed.
4. **Draft Update**:
   - Create or update relevant summary/entity notes under [03-Resources](file:///home/redmane/Documents/SecondBrain/03-Resources) or [01-Projects](file:///home/redmane/Documents/SecondBrain/01-Projects).
   - Add a one-line summary at the top of the new/updated note.
   - Cross-link the note to existing pages.
5. **Interactive Questioning**: If the source contradicts existing notes or contains ambiguous information, stop and present targeted clarification questions to the user.
6. **Finalize & Log**: Update the index and append a log entry to [CHANGELOG.md](file:///home/redmane/Documents/SecondBrain/CHANGELOG.md).

### B. Query Workflow
*Triggered when the user asks questions about vault contents.*

1. **Consult index.md First**: Read the master hub [index.md](file:///home/redmane/Documents/SecondBrain/index.md) to identify relevant entry points.
2. **Drill Down**: Read specific files identified in the index.
3. **Synthesize**: Formulate the response with clear wikilink citations (e.g. `[[index|My-Note]]`).
4. **Capture Insights**: If the synthesis represents a new useful connection, ask the user if they'd like to save it as a new note in the vault.

### C. Lint Workflow (Health Check & Gap-Finding)
*Triggered on command (e.g., "Run a full lint pass").*

Scan the entire vault and flag/fix:
1. **Broken/Missing Links**: Wikilinks pointing to non-existent notes.
2. **Orphan Notes**: Notes with no incoming links.
3. **Contradictions & Stale Claims**: Outdated project states or contradictions.
4. **Duplicate/Redundant Ideas**: Repeated concepts or rediscoveries.
5. **Unprocessed Inbox Items**: Stale items in [00-Inbox](file:///home/redmane/Documents/SecondBrain/00-Inbox) or [Clippings](file:///home/redmane/Documents/SecondBrain/Clippings).
6. **Knowledge Gaps**: Missing project docs or core topics mentioned but unwritten.
7. **Inconsistent Style**: Missing frontmatter, tags, or one-line summaries.
8. **Suggestions**: Propose new links, pages, or questions to ask the user.

### D. Vault Librarian Interviewer Routine
*Triggered on demand when the user requests to "run an interview session", "run a librarian session", or "populate the vault".*

1. **Role Activation**: Activate the `vault-librarian-interviewer` skill.
2. **Dynamic Generation**: Run `node scripts/vault-librarian.js` to scan the vault and generate 3–7 questions.
3. **Conduct Interview**: Present the questions. Capture Victor's answers conversationally.
4. **Ingest & Document**: Create/update notes in relevant PARA folders (never leave permanent notes in `00-Inbox`). Append summaries, YAML frontmatter, and bidirectional links.
5. **Persistent History**: Log the answered IDs using `node scripts/vault-librarian.js --record "id1,id2,..."`.
6. **Workflow Separation & Cadence**:
   * *Ingest Workflow*: Deals with capturing external information/clips.
   * *Lint Workflow*: Performs background health check audits and compiles `LINT-REPORT.md`.
   * *Interviewer Routine*: Facilitates direct personal knowledge elicitation from Victor.
   * *Cadence*:
     * *Daily*: Quick check-in on daily focus and blockers.
     * *Weekly*: Run full deep session on Sundays to clear backlogs, project gaps, and orphans.
     * *On-Demand*: Victor can invoke this anytime to record thoughts.

## 4. The 9-Doc Standard for Projects

Active projects under [01-Projects](file:///home/redmane/Documents/SecondBrain/01-Projects) should follow this standardized document structure:
1. **`PRD.md`**: Product Requirements Document. The primary entry point pointing to other project files.
2. **`TRD.md`**: Tech Stack, Architecture Decisions, and Constitutions.
3. **`PAGE_SPECS.md`**: Page layouts and functions.
4. **`APP_FLOW.md`**: Complete user journey and navigation paths.
5. **`UI/UX_BRIEF.md`**: UI style guides, color systems, micro-interactions, and component behavior.
6. **`SCHEMA.md`**: Database schemas, models, and data flow.
7. **`PHASES.md`**: Implementation roadmap split into clear verification steps.
8. **`NOTES.md`**: Journey log, daily developer logs, and open issues (renamed to avoid collisions on case-insensitive filesystems).
9. **`README.md`**: Public-facing overview generated/updated using the `readme-generator` skill.
