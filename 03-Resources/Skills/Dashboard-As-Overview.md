# Lesson: Dashboard As Overview

> **One-line Summary**: Dashboard/home is a calm overview and launchpad — not a dump of every feature; layout and top chrome guide the next action.

## The Trap
Overwhelming dashboards that try to be the whole app. Scanty dashboards that show nothing useful. No clear paths into sections.

## The Fix
1. **Job:** quick picture of "what's going on" + guidance to the right place.
2. **Not the job:** complete feature surface or every chart at full density.
3. Interface/layout decide whether users actually proceed into the work.
4. Common successful pattern: **top bar + sidebar** — top bar for quick access/title/actions, sidebar for primary destinations (roles can swap by product).
5. Cross-link with [[03-Resources/Skills/Sticky-Top-Bar-Navigation|Sticky Top Bar]] — always have a top bar; its role is product-specific.

### One rule
**Dashboard = overview + paths, not a second entire app.**

## Code Pattern / Prompt Template
```
/today (or dashboard):
- Summary cards/metrics (limited, high signal)
- Clear CTAs into sections (habits, fitness, log expense, …)
- Persistent shell: top bar + primary nav
- Avoid stuffing every module widget by default
```
