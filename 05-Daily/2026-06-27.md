# Tempire Learnings Summary — Vault Context

A brief record of developer learnings, structural observations, and architectural traps from the Tempire codebase.

---

### 1. Legal & Security Defensibility (Medium Priority)
*   **Observation:** Vibe-coded templates and MVP releases often neglect basic terms of service, privacy compliance, and RLS lockdown.
*   **Future Policy:** Ensure all projects include a standard template for Privacy Policy, Terms of Service, and a mandatory Supabase RLS security audit before showcase deploy. Victor will research and provide useful docs later

### 2. The Webhook Trap & Fix
*   **Trap:** Relying on API webhooks (SVIX) for immediate role sync on authentication guards creates race conditions; redirects to dashboards happen before the webhook database writes finish.
*   **Fix:** Use Clerk’s Custom Session Token JWT template to inject roles (`sessionClaims.role`) and invoke `await user?.reload()` on the client to force immediate propagation.

### 3. The Zoom Trap & Future Fixes
*   **Trap:** Designing a UI at 80% zoom visualizes density nicely but makes elements, spacing, and containers (`max-w-7xl`) appear bloated/oversized at standard 100% user zoom.
*   **Fix:** Constrain layout templates to narrower containers (e.g., `max-w-5xl` or `max-w-6xl`) and use smaller padding increments (`p-4` to `p-6`) at 100% zoom.

### 4. Download Security & Cart
*   **Observation:** The download system was designed to serve files using descriptive product names rather than database IDs, but left the raw URL exposed.
*   **Fix:** Secure downloads by proxying the request through a Next.js route handler that checks the paid order state in Supabase, generating a temporary Supabase Signed URL.
*   **Cart Tip:** Always use hydration state checks (e.g., `mounted` status) when reloading local storage cart items to prevent hydration mismatches.

### 5. File & Folder Organization (Tempire Slop)
*   **Observation:** Tempire contains redundant config/metadata structures (like two active `.agent` subfolders) and empty placeholder files at root (`car.`, `motorcycle`, `scooter`).
*   **Fix:** Establish strict linting or directory structure boundaries to prevent folder bloat and orphaned placeholder files.

### 6. Obsidian Vault Input Fix (Recurring issue)
*   **Observation:** I noticed in my SecondBrain vault's [README](README.md), the style of writing links "[[BRAIN|BRAIN.md]], [[AGENTS|AGENTS.md]], [[index|index.md]], and [[LINT-REPORT|LINT-REPORT.md]]" displays raw on GitHub. The links don't hide behind the text.
*   **Fix:** This format "[PRD](01-Projects/Tempire/Docs/PRD) | [TRD](01-Projects/Tempire/Docs/TRD) | [PAGE_SPECS](01-Projects/Tempire/Docs/PAGE_SPECS)" works better across many previews, on github, in a chat, even on mobile, when writing notes, etc.
*   **Recommendation:** Append rule to `AGENTS.md` to not use the initial link format. To instead use square brackets and parenthesis to display links. The links hide every where, except in their raw files.
